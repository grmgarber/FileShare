Show the ip address of the docker instance as seen on the host
    docker inspect -f '{{range .NetworkSettings.Networks}}{{.IPAddress}}{{end}}' <instance_id_or_name>

Enter into a running instance
    docker exec -it <instance_id_or_name> /bin/bash

Build all images
    docker-compose build

Start all instance
    docker-compose up -d   [-d is optional daemon mode]

Stop all instances
    docker-compose down

List all instances, running and finished
    docker ps -a

Kill a finished instance
    docker rm <instance_id>

List all images
    docker images

Remove an image
    docker rmi <image_id>

Run a one-off command inside the instance (example)
    docker-compose run --rm --workdir=/home/app/fupl webapp bin/rails --version

    --rm                - remove the container after the run
    --workdir           - working directory inside the container
    webapp              - name of the service drom the docker-compose.yml
    bin/rails --version - command executed in the container

        If more than one command needs to be excuted, run
    docker exec -it .... and then execute a series of commands

 e.g, to create the test database:
    docker-compose run --rm webapp bash -c "RAILS_ENV=test bundle exec rake db:create"
      to run the tests:
    docker-compose run --rm webapp bash -c "bundle exec rake spec"
